# BNIX Webmail

Self-hosted webmail client built with Python FastAPI. Supports IMAP/SMTP email, CalDAV calendar, and CardDAV contacts.

## Features

- 📧 Email — IMAP/SMTP with folders, search, attachments
- 📅 Calendar — CalDAV events with monthly grid view
- 👥 Contacts — CardDAV with search and CRUD
- ✍️ Signature — HTML editor with per-account settings
- 🔒 Session encryption — AES-256-GCM with HttpOnly cookies
- 📱 Responsive — Mobile-first design with Tailwind CSS

## Requirements

- **Python 3.10+**
- **Caddy** (reverse proxy with automatic HTTPS)
- **Linux server** (Ubuntu 24.04 / Debian 12+ recommended)

## Quick Start (Production)

```bash
# 1. Clone repo
git clone https://github.com/bnixvn/webmail.git /opt/bnix-webmail/src
cd /opt/bnix-webmail/src

# 2. Run installer
sudo bash deploy/linux/install.sh
```

The installer will:
- Install Python 3 + system packages
- Create `bnix-webmail` system user
- Setup Python venv and install dependencies
- Generate environment file at `/etc/bnix-webmail.env`
- Install and start systemd service

### Post-Install Configuration

Edit the environment file:

```bash
sudo nano /etc/bnix-webmail.env
```

```env
AUTH_SECRET=<auto-generated>

# ── Mail Server (Optional Override) ──
# By default, auto-discovers via DNS (SRV → common hostnames → MX).
# Only set these if you need to force a specific server for ALL domains.
IMAP_HOST=
IMAP_PORT=993
IMAP_SECURE=true
SMTP_HOST=
SMTP_PORT=465
SMTP_SECURE=true

# ── CalDAV / CardDAV (Optional) ──
DAV_HOST=
DAV_PORT=2080
DAV_SECURE=false

HOST=127.0.0.1
PORT=8000
DATA_DIR=/opt/bnix-webmail/data
```

Restart the service:

```bash
sudo systemctl restart bnix-webmail
```

> **Multi-domain support**: Users can log in with any email address. The app resolves the MX record for each email domain to find the correct IMAP/SMTP server. No per-domain configuration needed. If auto-detection fails, users can enter the server manually via "Mail server settings" on the login page.

### Caddy Reverse Proxy

Install Caddy and configure:

```bash
sudo apt install -y caddy
```

Copy the Caddyfile:

```bash
sudo cp backend/Caddyfile /etc/caddy/Caddyfile
sudo systemctl reload caddy
```

Or create `/etc/caddy/Caddyfile` manually:

```
webmail.yourdomain.com {
    reverse_proxy 127.0.0.1:8000
}
```

## Local Development

```bash
# Setup Python venv
python3 -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements.txt

# Create .env
cp .env.example .env
# Edit .env with your IMAP/SMTP settings

# Run
cd backend
python main.py
```

Open http://localhost:8000

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/login` | Login with email + password |
| POST | `/api/auth/logout` | Logout |
| GET | `/api/auth/me` | Check session (public — returns `{authenticated: false}` if not logged in, no 401) |
| GET | `/api/mailboxes` | List mailboxes |
| GET | `/api/messages` | List messages |
| GET | `/api/messages/{uid}` | Get message |
| DELETE | `/api/messages/{uid}` | Delete message |
| PATCH | `/api/messages/{uid}/flags` | Update flags |
| POST | `/api/messages/{uid}/move` | Move message |
| POST | `/api/messages/send` | Send email |
| GET | `/api/calendar` | List events |
| POST | `/api/calendar` | Create event |
| PUT | `/api/calendar/{uid}` | Update event |
| DELETE | `/api/calendar/{uid}` | Delete event |
| GET | `/api/contacts` | List contacts |
| POST | `/api/contacts` | Create contact |
| PUT | `/api/contacts/{uid}` | Update contact |
| DELETE | `/api/contacts/{uid}` | Delete contact |
| GET | `/api/avatar` | Get avatar (BIMI/Gravatar) |
| GET | `/api/settings/signature` | Get signature |
| PUT | `/api/settings/signature` | Update signature |
